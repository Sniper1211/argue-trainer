# 项目简报 - Argue Trainer

## 📋 项目概述

**Argue Trainer** 是一个基于AI的理性思辨与情绪控制训练平台，旨在帮助用户在冲突场景中提升沟通能力。

## 🎯 核心价值主张

### 用户痛点
- 不善言辞，被语言挤兑后反击发挥不好
- 冲突后复盘懊恼，缺乏有效训练方法
- 需要提升理性思辨与情绪控制能力

### 解决方案
- AI模拟真实冲突对话场景
- 逻辑谬误识别与反驳训练
- 情绪状态反馈与脱敏训练
- 场景化策略选择指导

## 🏗️ 技术架构

### 前端架构
```
Next.js 14 (App Router)
├── 页面路由 (app/*)
├── 组件库 (components/*)
├── 状态管理 (Zustand)
├── 样式系统 (Tailwind CSS)
└── API客户端 (DeepSeek集成)
```

### 架构设计（纯前端 + API代理）
```
Next.js App (Vercel托管)
├── 前端页面 (app/*)
├── API Routes (app/api/*) - 仅用于AI API代理
└── 无数据库 - 本地存储或逐步添加
```

### 数据流（成本优化版）
```
用户输入 → 前端界面 → Next.js API Route（代理） → DeepSeek API → 返回前端
      ↑
  本地存储（localStorage）← 训练历史缓存
```

## 🚀 MVP功能清单

### 核心功能
1. **基础对话界面**
   - 类ChatGPT对话UI
   - 消息历史记录
   - 场景选择面板

2. **AI对话引擎**
   - DeepSeek API集成
   - 场景化prompt模板
   - 逻辑分析中间件

3. **训练场景库**
   - 职场冲突 (3个场景)
   - 亲密关系 (2个场景) 
   - 消费维权 (2个场景)

4. **基础反馈系统**
   - 逻辑谬误识别
   - 简单情绪反馈
   - 策略建议

### 技术实现要点
1. **AI集成优化**
   - 系统prompt设计
   - 上下文管理
   - 响应格式化

2. **用户体验优化**
   - 加载状态处理
   - 错误边界
   - 离线支持

3. **性能考虑**
   - API响应缓存
   - 消息流式传输
   - 代码分割

## 🎨 设计原则

### 用户体验
- **简洁直观**：降低学习成本
- **即时反馈**：实时AI响应
- **渐进式**：从简单到复杂场景

### 技术实现
- **模块化**：组件可复用
- **可扩展**：易于添加新功能
- **可维护**：清晰代码结构

### 产品理念
- **教育性**：注重能力提升而非对立
- **实用性**：真实可用的训练工具
- **包容性**：适合不同沟通风格用户

## 🤔 风险与应对

### 技术风险
- **AI响应质量不稳定**
  - 应对：优化prompt，添加后处理
- **API成本控制**
  - 应对：缓存策略，使用限制

### 产品风险
- **用户需求验证不足**
  - 应对：MVP快速验证，用户访谈
- **使用场景有限**
  - 应对：扩展场景库，用户共创

### 运营风险
- **用户获取困难**
  - 应对：内容营销，社群运营
- **用户粘性不足**
  - 应对：成就系统，个性化推荐
